天气预报查询系统项目说明
项目介绍
本项目是一款轻量级前端天气预报查询网页应用，全程采用纯前端技术开发，无需后端服务器与数据库支持。项目依托 GitHub Pages 实现静态网页免费线上部署，用户可通过浏览器随时访问使用。平台支持国内城市实时天气查询，能够展示温度、天气状况、风向风力、空气湿度等气象信息，并具备本地缓存记忆功能，使用体验简洁流畅，同时适配电脑端与移动端设备。
核心技术概述
1. 前端基础技术
本项目基于标准前端三件套完成开发，包含 HTML5、CSS3 与原生 JavaScript，无任何框架依赖，项目轻量化、加载速度快、兼容性强。HTML5 用于搭建整体页面结构，划分搜索模块与天气展示模块，构建完整的页面骨架；CSS3 负责页面样式美化与自适应布局，实现渐变背景、卡片样式、按钮交互效果，同时适配手机、电脑等不同设备屏幕；JavaScript 作为核心交互脚本，实现页面所有动态功能，包含用户监听输入、查询触发、数据请求、内容渲染和异常提示等逻辑。
2. 数据请求与处理技术
项目采用浏览器原生 Fetch 接口实现异步网络请求，无需第三方请求库，通过调用第三方公开气象 API 获取标准化天气数据。接口可根据输入的城市名称，返回实时气象数据，前端对返回的 JSON 数据进行解析、筛选与渲染，完成天气信息的页面展示。同时使用浏览器 LocalStorage 本地存储技术，自动保存用户上一次查询的城市信息，实现页面重载后自动恢复历史查询记录的效果。
3. 线上部署技术
项目使用 GitHub Pages 进行静态网站托管服务，该服务专门用于部署纯前端静态资源，支持 HTML、CSS、JavaScript 文件在线分发。开发者将项目源码上传至 GitHub 公开仓库后，开启 Pages 部署功能，即可生成永久公开的在线访问地址，部署零成本、稳定性高、更新便捷。
项目运行原理
页面初始化时，系统优先读取浏览器本地缓存数据，自动加载用户上次查询的城市天气。用户可手动输入新的城市名称，通过点击查询按钮或敲击回车键触发查询事件。前端脚本接收用户输入信息，通过异步请求调用气象开放接口，获取对应城市的实时天气数据。系统对接口返回的数据进行合法性校验，数据正常则动态渲染天气信息至页面，数据异常或网络错误时则展示对应的错误提示信息。查询成功后，系统自动更新本地缓存，保存当前查询城市，实现持久化记忆功能。
部署使用说明
在 GitHub 新建公开仓库，将项目静态文件上传至仓库主分支。进入仓库设置界面，开启 GitHub Pages 服务，选择主分支与根目录作为资源部署路径，等待部署完成后即可生成在线访问链接。后续修改页面内容后，只需推送最新代码至仓库，线上网站即可自动同步更新。
注意事项
项目天气数据来源于第三方开放 API，使用前需要自行申请有效接口密钥并配置生效，否则无法正常获取天气数据。本项目为纯静态前端项目，无后端中转服务，部分网络环境下可能存在接口跨域限制。项目仅支持静态资源运行，不支持后端程序、数据库等服务运行。
