一、项目概述
本项目是基于纯前端开发的天气预报查询单页应用，无后端服务，依托 GitHub Pages 实现免费线上部署，访问地址：https://dengxixin12.github.io/laoto/。用户输入城市名称即可查询实时温度、天气状况、风力、湿度等气象信息，支持本地缓存历史查询城市，适配电脑与移动端设备。
二、项目关键技术说明
（一）页面基础构建技术
HTML5
搭建页面语义化骨架，划分搜索输入区域、天气展示卡片区域，定义交互 DOM 节点，作为页面承载基础。
CSS3
使用 Flex 弹性布局实现自适应排版，设置渐变背景、圆角卡片、半透明遮罩、按钮交互样式，通过基础适配规则完成移动端兼容，无需第三方样式框架，轻量化运行。
JavaScript ES6+
实现页面全部交互逻辑，包含按钮点击监听、回车快捷查询、异步接口请求、JSON 数据解析、动态 DOM 渲染、异常捕获、本地存储等核心业务功能。
（二）气象数据获取技术
Fetch API
原生异步网络请求工具，调用第三方和风天气开放 API，获取城市实时气象 JSON 数据，搭配 async/await 简化异步代码逻辑。
第三方天气开放 API
项目数据源，传入城市名称与专属密钥，返回实时气温、天气文字、风向风力、空气湿度等标准化 JSON 数据。
LocalStorage 本地存储
持久化存储用户上一次查询的城市名称，页面打开自动读取缓存并自动查询天气，优化用户使用体验。
（三）线上部署技术：GitHub Pages
仅支持 HTML、CSS、JS 静态文件，无需服务器、数据库；
将项目源码上传至 GitHub 代码仓库，开启 Pages 服务后自动生成公开访问网址；
免费、稳定、实时更新，推送代码后页面自动同步线上内容。
四、GitHub 上传与部署操作步骤
1. 仓库文件准备
本地新建文件夹 laoto；
将上方完整代码复制保存为 index.html，放入文件夹内（仅需这一个文件，样式、脚本已内联，无需额外 css/js 文件）。
2. GitHub 上传流程
登录github.com，新建仓库，仓库名填写 laoto，仓库属性设为 Public；
进入仓库，点击 Add file - Upload files，将本地index.html上传提交；
提交填写备注：初始化天气预报项目，点击 Commit changes 完成上传。
3. 开启 GitHub Pages 线上访问
仓库顶部点击 Settings（设置）；
侧边栏找到 Pages；
Source 来源选择 Deploy from a branch，分支选择 main，文件夹选择 /(root)；
保存设置，等待 1-2 分钟自动生成线上地址：https://dengxixin12.github.io/laoto/。
五、项目运行流程说明
页面加载：读取本地缓存的历史城市，自动填充输入框并查询天气；
用户输入城市，点击查询按钮或按下回车键触发查询函数；
JS 通过 Fetch 请求天气 API，传输城市名称与密钥；
接收接口返回的 JSON 气象数据，校验请求状态；
数据正常则动态生成天气卡片渲染至页面，异常则提示错误信息；
查询成功后自动将城市存入 LocalStorage，下次打开页面自动读取。
六、注意事项
代码中 API_KEY 需要前往和风天气官网注册账号，免费申请专属密钥后替换，否则无法获取气象数据；
GitHub Pages 仅托管静态文件，不支持后端接口，全部数据请求依靠第三方开放 API；
若出现跨域报错，可更换支持跨域的气象接口或使用在线 CORS 代理；
修改代码后重新上传仓库，线上页面会自动更新。
