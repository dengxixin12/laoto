一、项目简介
本项目是纯前端天气预报查询网页，无后端服务，依托 GitHub Pages 实现免费线上部署，可直接通过浏览器访问。用户输入城市名称，即可查看对应城市实时气温、天气状况、风向风力、空气湿度等气象信息；页面具备本地记忆功能，会保存上一次查询的城市，再次打开页面自动加载历史天气，同时适配电脑、手机等不同尺寸设备。
二、项目核心关键技术
（一）前端页面基础技术
HTML5
作为页面结构搭建基础，划分搜索输入区域、天气信息展示区域两大核心模块，搭建完整页面骨架，提供所有交互控件与内容承载容器，满足页面基础展示需求。
CSS3
负责页面视觉美化与自适应布局，采用弹性布局完成页面元素排版，搭配渐变背景、圆角卡片、半透明效果优化视觉观感；设置按钮悬停交互样式，同时添加移动端适配规则，保证手机端正常浏览。
JavaScript ES6+
项目交互逻辑核心，承担页面全部动态功能。包含监听按钮点击、键盘回车快捷查询、网络数据请求、接口返回数据解析、页面内容动态刷新、异常捕获提示、本地数据存储等全套业务逻辑。
（二）气象数据获取相关技术
Fetch 原生网络请求
采用浏览器内置异步请求工具对接第三方气象开放接口，搭配异步语法简化网络请求流程，实现无需刷新页面即可获取天气数据。
第三方气象开放 API
项目唯一数据源，通过传递城市名称与专属访问密钥，接口返回标准化 JSON 格式气象数据，包含实时温度、天气描述、风力风向、湿度等气象参数。
LocalStorage 本地持久化存储
利用浏览器本地存储功能，记录用户上次查询的城市名称，页面初始化时自动读取缓存内容并发起天气查询，提升重复使用体验。
（三）线上部署技术：GitHub Pages
该技术用于静态网页线上托管，仅支持 HTML、CSS、JavaScript 静态资源，无需搭建服务器与数据库。将项目文件上传至 GitHub 代码仓库后，开启 Pages 功能即可生成公开网络访问地址，代码更新推送后线上页面会自动同步，完全免费使用。
三、整体业务运行流程
页面初始化加载阶段：程序读取浏览器本地存储中保存的历史城市，自动填充输入框并发起天气查询；若无历史记录，则展示空白查询界面。
用户操作阶段：在输入框填写目标城市，点击查询按钮或按下回车键，触发天气查询逻辑。
网络请求阶段：程序将城市名称与访问密钥拼接，向第三方气象接口发送异步请求。
数据校验与渲染阶段：接收接口返回数据，校验请求状态；请求成功则提取各类气象信息，自动生成可视化卡片展示在页面；请求失败、城市不存在或网络异常时，页面弹出对应文字提示。
缓存保存阶段：查询成功后，自动将当前城市名称存入浏览器本地存储，供下次打开页面调用。
四、GitHub 上传与部署操作说明
文件准备：整理项目所有静态页面文件，统一放置在同名项目文件夹中。
仓库创建：登录 GitHub 官网，新建公开代码仓库，仓库名称与项目文件夹保持一致。
文件上传：进入新建仓库，选择上传文件功能，将本地项目全部文件提交至仓库主分支。
开启线上访问服务：打开仓库设置页面，找到 Pages 配置项，设置部署来源为仓库主分支、根目录保存资源，保存后等待片刻即可生成专属线上访问链接。
五、项目使用注意事项
气象接口密钥需要自行前往对应气象开放平台注册账号申请，未替换有效密钥将无法获取天气数据。
本项目为纯静态前端项目，无后端中转服务，全部气象数据直接由前端向第三方接口发起请求，部分环境下可能出现跨域访问限制。
仅支持静态资源托管，无法运行后端程序、数据库等服务；修改页面内容后，需重新上传文件至 GitHub 仓库，线上网页才会更新。
